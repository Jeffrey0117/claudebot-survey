# 躺在床上用語音，兩小時做完一個網站 — 我的 AI 開發實錄

## 前言：這不是教學，是一場實驗

大家好，我是 html_cat，切版職人。

這篇文章要講的事情有點瘋：我躺在床上，用手機語音輸入，兩小時內從零做出一個完整的課程興趣調查網站，包含前台問卷、折扣券、後台數據儀表板，而且直接上線到 survey.isnowfriend.com。

沒開電腦。沒碰鍵盤。全程用嘴巴寫 code。

聽起來很唬爛對吧？讓我從頭說起。

---

## 起因：我需要一個問卷網站

最近在規劃一堂「用 AI 打造個人開發工作流」的課程，想先做個興趣調查，看看大家對哪些主題有興趣、願不願意用訂閱制。

一般人可能會開 Google Forms 五分鐘搞定。但我想：既然要教人用 AI 開發，那我自己的問卷網站，是不是也該用 AI 做出來？而且最好是用最懶的方式 — 在手機上完成。

於是我打開 Telegram，跟我的 ClaudeBot 說：

> 「我想做一個課程問卷網站，讓人填興趣調查，技術選型你建議什麼？」

AI 秒回：Next.js 15 + App Router + Tailwind CSS + TypeScript。OK，就這樣開始了。

---

## 第一個坑：Google 登入搞不定

一開始我想做 Google OAuth 登入，讓填問卷的人先登入再填。結果 Google OAuth consent screen 審核卡住，各種設定繞來繞去。

在手機上 debug OAuth 真的很痛苦，我想了三秒鐘，直接語音：

> 「算了，不要要求登入，直接讓人看到問卷就好。」

AI 立刻把 NextAuth 整套拔掉，改成免登入流程。這就是用 AI 開發的好處 — 砍掉重練的成本趨近於零。你不用心疼寫過的 code，因為那不是你打的。

---

## 語音輸入 = 最快的需求文件

接下來進入瘋狂迭代階段。我把需求一句一句用語音丟出去，每一句話 AI 就改一版，自動 build、commit、push，然後透過 CloudPipe webhook 自動部署上線。

這些是我實際講的話：

> 「姓名欄位改成 Threads 帳號，這樣我比較好聯絡。」

> 「價格的題目拿掉，我自己定價就好，不用問他們。」

> 「上面要介紹一下我是 html_cat 切版職人，讓人知道是誰辦的。」

> 「用我的大頭貼，去 classroo.tw 上面抓。」

> 「填完之後送折扣券，代碼 BOT200 折兩百塊。」

> 「上課方式那題改成訂閱制的意願調查。」

> 「每個欄位底下加一行小字說明為什麼要填這個。」

每一句話，從語音辨識到 AI 理解到改 code 到部署上線，大概就是一兩分鐘的事。我甚至不用看程式碼，直接在手機瀏覽器重新整理就能看到結果。

不滿意？再講一句就好。

## UI 重寫：深色主題 + 折扣券動畫

初版的 UI 太陽春了，我請 AI 用 frontend-design 模式重新設計整個頁面。

最後的成品是深色主題（背景色 `#0b0d14`），搭配 JetBrains Mono 字體，整體走一個開發者調性的風格。畢竟目標受眾就是工程師和設計師嘛。

折扣券的部分我特別喜歡 — 卡片有鋸齒邊效果（像真的優惠券），加上 shimmer 光澤動畫和 pulse 呼吸燈效果。填完問卷看到那張券彈出來，體驗蠻爽的。

Radio button 和 checkbox 也全部客製化了，不是瀏覽器預設的醜樣式。OG meta tags 也設好了，分享到社群會帶折扣資訊的預覽圖。

這些全部都是語音指揮完成的，我只負責「驗收」— 在手機上看順不順眼。

## 後台也要：Admin Dashboard

問卷做完了，但我還需要看數據啊。

> 「後台要能清楚看數據，幫我做一個 admin dashboard。」

AI 做了一個密碼驗證的後台頁面（不需要 Google 登入，打密碼就好），裡面有三個 tab：

**統計總覽** — 頂部一排快速數據卡片，一眼看到總回覆數、興趣比例、訂閱意願比例。下面是彩色長條圖，每個選項的統計一目了然。

**聯絡人列表** — 所有填過問卷的人，可以一鍵複製全部 Email 或 Threads 帳號。要發通知的時候直接貼上就好。

**文字回饋** — 開放式問題的回答全部列出來，方便我看大家的想法。

資料存在 JSON 檔案裡（`data/surveys.json`），沒用資料庫。對這個規模的問卷來說，JSON 檔案完全夠用，簡單粗暴。

## 部署流程：一句話上線

這邊稍微講一下技術架構，因為這也是整個體驗這麼順的關鍵。

每次 AI 改完 code，流程是這樣的：

1. **Build** — `next build` 確認沒有編譯錯誤
2. **Commit + Push** — 推到 GitHub（Jeffrey0117/claudebot-survey）
3. **CloudPipe Webhook** — GitHub 推送觸發 CloudPipe 自動部署
4. **PM2 重啟** — 新版本自動上線
5. **Cloudflare Tunnel** — `survey.isnowfriend.com` 路由到伺服器

整條鏈路全自動。我在手機上講完一句需求，等個一兩分鐘，重新整理瀏覽器就是新版本了。

中間有一個小插曲：第一次部署完，我在家裡 Wi-Fi 打不開網站。慌了一下，切 4G 發現正常 — 原來是路由器 DNS 緩存了 NXDOMAIN。等快取過期就好了。這種小事也是 AI 幫我排查的。

## 心得：AI 開發的門檻已經變了

這次經驗讓我確信一件事：**AI 開發的瓶頸不是技術能力，是你能不能把需求講清楚。**

你不需要會寫 Next.js。你不需要會設定 Tailwind。你甚至不需要坐在電腦前面。你需要的是知道自己要什麼，然後用人話把它說出來。

當然，這背後有 ClaudeBot 的基礎建設在支撐 — 語音辨識、Claude Code CLI、自動部署流水線。這些工具鏈讓「語音到上線」這條路變得可行。

這也是我想開這堂課的原因。不是教你寫 code，是教你怎麼建立自己的 AI 開發工作流，讓你也能用這種方式做事。
